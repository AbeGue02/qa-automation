# Software Testing / QA Automation

### How can I ensure that my code, program, or software is functional and secure?

To ensure that any software I am developing in functional and secure, I must implement differents forms of software testing and automation. The main strategy to ensure code works is **Unit Testing**, which ensures that individual, isolated sections of code work. After that, I must implement **Integration testing** to make sure these different objects work when used together. As more parts of the program are being developed, I can use **Regression Testing** to make sure new changes have no broken old features. Through this, and many more strategies, I can identify when a program works and is secure.

### How do I interpret user needs and incorporate them into a program?

When interpreting user needs, it is very important to make sure their needs are implemented in the program being developed. By using user stories, I should be able to make sure every single user need is taken into account. Additionally, using software testing, I can write tests for specific requirements found during the requirement elicitation process to further ensure the development team is building the right product. 

### How do I approach designing software?

When designing software, I make sure to break tasks down into smaller tasks and taking everything one step at a time. I make sure to follow the core principles of designing software and always keeping in mind that the software must be scalable, it must protect the integrity of the data, and it must be well documented over everything else.
